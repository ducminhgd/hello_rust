## Variable

1. An immutable variable `let x = 42;`
2. A mutable variable `let mut x = 42;`
3. Constant `const MY_CONSTANT = 42;`
4. Type hinting for variable `let x: u32 = 42;`
5. Shadowing

    ```rust
    fn main() {
        let x = 5;

        let x = x + 1;

        {
            let x = x * 2;
            println!("The value of x in the inner scope is: {x}");
        }

        println!("The value of x is: {x}");
    }
    ```

## Statements and Expressions

1. Not OK
     
     ```rust
     fn main() {
         let x = (let y = 6);
     }
     ```

2. OK
     
     ```rust
     fn main() {
         let y = {
             let x = 3;
             x + 1
         };

         println!("The value of y is: {y}");
     }
     ```
 
 3. Function returns value

     ```rust
     fn five() -> i32 {
         5
     }

     fn main() {
         let x = five();

         println!("The value of x is: {x}");
     }

     // OR

     fn main() {
         let x = plus_one(5);

         println!("The value of x is: {x}");
     }

     fn plus_one(x: i32) -> i32 {
         x + 1
     }
    ```

## `const` vs `let`

Differences: the initializing time, storage place, and how the compiler optimizes.

### `const`

```rust
const UNITS: [&str; 5] = ["B", "KiB", "MiB", "GiB", "TiB"];
```

- Evaluated entirely at compile time.
- Inlined at each use site.

    ```rust
    //  In rust
    println!("{}", UNITS[0]);
    println!("{}", UNITS[1]);

    // when compile
    println!("{}", ["B", "KiB", "MiB", "GiB", "TiB"][0]);
    println!("{}", ["B", "KiB", "MiB", "GiB", "TiB"][1]);
    ```

- May not occupy dedicated memory after optimization.
- Ideal for immutable compile-time values.

### `let`

```rust
let units = ["B", "KiB", "MiB", "GiB", "TiB"];
```

- Created at runtime.
- The array is allocated on the stack for each function call.
- Each element is a fat pointer (`&str` = pointer + length)
- `&str` elements still point to string literals stored in `.rodata`.

### `static`

```rust
static UNITS: [&str; 5] = ["B", "KiB", "MiB", "GiB", "TiB"];
```

- Exactly one instance exists for the entire program.
- Stored in static memory (`.rodata` for immutable data).
- Suitable for large shared lookup tables or when a stable address is required.

Typical use cases:

- Large lookup tables
- Shared immutable data
- Stable memory address
- Global mutable state (rare)

### `const` vs `static`

| Use case | `const` | `static` |
|----------|---------|----------|
| Compile-time constant | ✅ | ✅ |
| Single shared instance | ❌ | ✅ |
| Stable memory address | ❌ | ✅ |
| Large lookup table | ⚠️ Prefer `static` | ✅ |
| Runtime mutable state | ❌ | ✅ (with synchronization) |


## Recommendation

Keep:

```rust
const UNITS: [&str; 5] = ["B", "KiB", "MiB", "GiB", "TiB"];
```

It best represents immutable compile-time data and allows the compiler to optimize aggressively.

Use `let` only when the value depends on runtime conditions.

Use `static` when a single shared instance or a stable memory address is required.
